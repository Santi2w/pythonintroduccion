3 Tipos de datos compuestos ( colecciones )

3.1 Listas

  • Una coleccin de objetos
  • Mutables
  • Tipos arbitrarios heterogeneos
  • Puede contener duplicados
  • No tienen tamao fijo. Pueden contener tantos elementos como quepan en memoria.
  • Los elementos van ordenados por posicin.
  • Se acceden usando la sintaxis: [index].
  • Los ndices van de 0 a n-1, donde n es el nmero de elementos de la lista.
  • Son un tipo de Secuencia, al igual que los strings; por lo tanto, el orden (es decir,la posicin de los objetos de la lista) es importante.
  • Soportan anidamiento.
  • Son una implementacin del tipo abstracto de datos: Array Dinmico.

  • Coleccin de parejas clave-valor.
  • Son mutables.
  • Claves:

3.3 Sets (Conjuntos)

Al igual que las listas:

  • Coleccin de elementos.
  • Tipos arbitrarios.
  • Mutables.
  • No tienen tamao fijo. Pueden contener tantos elementos como quepan en memoria.

A diferencia de las listas:

  • No puede tener duplicados.
  • Se definen por medio de llaves.
  • Los elementos no van ordenados por posicin. No hay orden establecido.
  • Solo pueden contener objetos inmutables.
  • No soportan anidamiento.

3.4 Tuplas

Al final que las listas:

  • Coleccin de elementos.
  • Tipos arbitrarios.
  • Puede contener duplicados.
  • No tienen tamao fijo. Pueden contener tantos elementos como quepan en memoria.
  • Los elementos van ordenados por posicin.
  • Acceso a travs de la sintaxis: [index]
  • ndices van de 0 a n-1, donde n es el nmero de elementos de la tupla.
  • Son secuencias donde el orden de los elementos importa.
  • Soportan anidamiento.

A diferencia de las listas:

  • Se definen por medio de parntesis.
  • Inmutables.

Por qu tuplas?

  • Representacin de una coleccin fija de elementos (por ejemplo, una fecha).
  • Pueden usarse en contextos que requieren inmutabilidad (por ejemplo, como claves de un diccionario).
